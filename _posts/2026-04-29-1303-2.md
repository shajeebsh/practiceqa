### 1\. Big Countries — Easy

Problem| Solution  
---|---  
A country is "big" if it has an area of at least 3,000,000 km² **or** a population of at least 25,000,000. Return `name`, `population`, `area` for all big countries. **DataFrame:** `world` with columns `name, continent, area, population, gdp`| Filter rows using boolean conditions on area and population  
      
    
    import pandas as pd
    
    def big_countries(world: pd.DataFrame) -> pd.DataFrame:
        return world[
            (world['area'] >= 3_000_000) | (world['population'] >= 25_000_000)
        ][['name', 'population', 'area']]
    
    
    

**Key concept:** Boolean indexing with `|` (OR). Use `&` for AND. Always wrap each condition in parentheses.

### 6\. Calculate Special Bonus — Easy

Problem| Solution  
---|---  
Pay each employee a bonus of 100% of their salary if: their `employee_id` is **odd** AND their name does **not** start with `'M'`. Otherwise bonus = 0. Return `employee_id` and `bonus` sorted by `employee_id`. **DataFrame:** `employees(employee_id, name, salary)`| Use `np.where` or a boolean mask with `.str.startswith()`  
      
    
    import pandas as pd
    import numpy as np
    
    def calculate_special_bonus(employees: pd.DataFrame) -> pd.DataFrame:
        mask = (employees['employee_id'] % 2 == 1) & (~employees['name'].str.startswith('M'))
        employees['bonus'] = np.where(mask, employees['salary'], 0)
        return employees[['employee_id', 'bonus']].sort_values('employee_id')
    
    
    

**Key concept:** `np.where(condition, value_if_true, value_if_false)` is the vectorised if-else. `.str.startswith('M')` checks prefix.

### 8\. Find Users With Valid E-Mails — Easy

Problem| Solution  
---|---  
Find users with valid emails. A valid email: starts with a letter, then letters/digits/underscore/period/dash, followed by `@leetcode.com`. **DataFrame:** `users(user_id, name, mail)`| Use `.str.match()` with a regex pattern  
      
    
    import pandas as pd
    
    def valid_emails(users: pd.DataFrame) -> pd.DataFrame:

**Regex breakdown:**
    
    
    ^           → start of string

    $           → end of string
    
    
    

**Key concept:** `.str.match(pattern)` applies regex anchored at the start. Use `.str.contains(pattern)` for unanchored search.

### 9\. Patients With a Condition — Easy

Problem| Solution  
---|---  
Find patients who have **Type I Diabetes** — a condition code starting with `DIAB1`. A patient may have multiple conditions separated by spaces. **DataFrame:** `patients(patient_id, patient_name, conditions)`| Use `.str.contains()` with a regex word-boundary pattern  
      
    
    import pandas as pd
    
    def find_patients(patients: pd.DataFrame) -> pd.DataFrame:
        # DIAB1 must appear as a word prefix: start of string or after a space
        return patients[patients['conditions'].str.contains(r'(?<!\S)DIAB1', regex=True)]

        mask = (patients['conditions'].str.startswith('DIAB1') |

### 10\. Count Occurrences in Text — Medium

Count how many times the word **"bull"** and the word **"bear"** appear as standalone words across all files. Return a result with `word` and `cnt`. **DataFrame:** `files(file_name, content)`| Use `.str.count()` with word-boundary regex  
      
    
    import pandas as pd

        bull_count = files['content'].str.count(r'\bbull\b').sum()

### 11\. Nth Highest Salary — Medium

Problem| Solution  
---|---  
Return the N-th highest **distinct** salary from the `employee` table. If it doesn't exist, return `None`. **DataFrame:** `employee(id, salary)`| Drop duplicates, sort descending, use `.iloc` with bounds check  
      
    
    import pandas as pd
    
    def nth_highest_salary(employee: pd.DataFrame, N: int) -> pd.DataFrame:
        unique_salaries = employee['salary'].drop_duplicates().sort_values(ascending=False)
        result = unique_salaries.iloc[N - 1] if N <= len(unique_salaries) else None
        col_name = f'getNthHighestSalary({N})'
        return pd.DataFrame({col_name: [result]})
    
    
    

**Key concept:** `.drop_duplicates()` removes repeated values. `.iloc[N-1]` accesses by 0-based position. Always guard with a length check to avoid `IndexError`.

### 13\. Department Highest Salary — Medium

Problem| Solution  
---|---  
Find employees who have the highest salary in their department. **DataFrames:** `employee(id, name, salary, departmentId)`, `department(id, name)`| GroupBy + transform to get max per group, then filter  
      
    
    import pandas as pd
    
    def department_highest_salary(employee: pd.DataFrame, department: pd.DataFrame) -> pd.DataFrame:
        # Get max salary per department
        employee['max_salary'] = employee.groupby('departmentId')['salary'].transform('max')
        # Keep only rows where salary == max
        top = employee[employee['salary'] == employee['max_salary']]
        # Join with department names
        result = top.merge(department, left_on='departmentId', right_on='id', suffixes=('_emp', '_dept'))
        return result[['name_dept', 'name_emp', 'salary']].rename(
            columns={'name_dept': 'Department', 'name_emp': 'Employee', 'salary': 'Salary'}
        )
    
    
    

**Key concept:** `.groupby().transform('max')` broadcasts the group max back to every row — essential for filtering within groups without losing rows.

### 14\. Rank Scores — Medium

Problem| Solution  
---|---  
Rank scores in descending order. Tied scores get the same rank, and ranks have no gaps (`DENSE_RANK`). **DataFrame:** `scores(id, score)`| Use `.rank(method='dense', ascending=False)`  
      
    
    import pandas as pd
    
    def order_scores(scores: pd.DataFrame) -> pd.DataFrame:
        scores['rank'] = scores['score'].rank(method='dense', ascending=False).astype(int)
        return scores[['score', 'rank']].sort_values('rank')
    
    
    

**Rank methods comparison:**
    
    
    method='dense'   → 1,1,2,2,3    ← what we want (no gaps)
    method='min'     → 1,1,3,3,5    (gaps after ties)
    method='average' → 1.5,1.5,3.5  (average of tied positions)
    method='first'   → 1,2,3,4,5    (no ties)

### 16\. Rearrange Products Table — Easy

Problem| Solution  
---|---  
Pivot a wide table (columns: `product_id, store1, store2, store3`) to a long table with columns `product_id, store, price`. Exclude rows where price is NULL. **DataFrame:** `products(product_id, store1, store2, store3)`| Use `pd.melt()` to unpivot from wide to long format  
      
    
    import pandas as pd
    
    def rearrange_products_table(products: pd.DataFrame) -> pd.DataFrame:
        result = products.melt(
            id_vars='product_id',
            value_vars=['store1', 'store2', 'store3'],

**Key concept:** `pd.melt()` converts wide → long format. `id_vars` = columns to keep as-is; `value_vars` = columns to unpivot; `var_name` = new column for old column names; `value_name` = new column for values.

### 20\. Ads Performance — Easy

Problem| Solution  
---|---  
Compute the **Click-Through Rate (CTR)** for each ad: `CTR = clicks / (clicks + views) * 100`, rounded to 2 dp. Ads with no clicks/views have CTR = 0. **DataFrame:** `ads(ad_id, user_id, action)` — action ∈ {Clicked, Viewed, Ignored}| Pivot counts by action, then compute CTR formula  
      
    
    import pandas as pd
    
    def ads_performance(ads: pd.DataFrame) -> pd.DataFrame:
        # Count clicks and views per ad
        grouped = ads.groupby(['ad_id', 'action']).size().unstack(fill_value=0)
        # Ensure columns exist
        for col in ['Clicked', 'Viewed', 'Ignored']:
            if col not in grouped.columns:
                grouped[col] = 0
        grouped['ctr'] = grouped.apply(
            lambda row: round(row['Clicked'] / (row['Clicked'] + row['Viewed']) * 100, 2)
            if (row['Clicked'] + row['Viewed']) > 0 else 0.0,
            axis=1
        )
        return grouped[['ctr']].reset_index()[['ad_id', 'ctr']].sort_values('ctr', ascending=False)

        exam_counts = examinations.groupby(
            ['student_id', 'subject_name'], as_index=False
        ).size().rename(columns={'size': 'attended_exams'})
    
        # Step 3: Left join so 0-attendance pairs are included
        result = cross.merge(exam_counts, on=['student_id', 'subject_name'], how='left')
        result['attended_exams'] = result['attended_exams'].fillna(0).astype(int)
    
        return result.sort_values(['student_id', 'subject_name'])[
            ['student_id', 'student_name', 'subject_name', 'attended_exams']
        ]
    
    
    

**Key concept:** `how='cross'` performs a Cartesian product (every row × every row). Then a left join + `fillna(0)` handles the zero-attendance cases.

### 31\. Managers with at Least 5 Direct Reports — Medium

Problem| Solution  
---|---  
Find managers who have **at least 5 direct reports**. Return their `name`. **DataFrame:** `employee(id, name, department, managerId)`| Count reports per managerId, filter >= 5, then join back to get name  
      
    
    import pandas as pd
    
    def find_managers(employee: pd.DataFrame) -> pd.DataFrame:
        # Count direct reports per manager
        report_counts = employee.groupby('managerId', as_index=False).size()
        # Filter managers with >= 5 reports
        qualified = report_counts[report_counts['size'] >= 5]
        # Join to get manager names (manager's id matches the managerId)
        result = qualified.merge(employee, left_on='managerId', right_on='id')
        return result[['name']]
    
    
    

**Key concept:** Self-referencing join — the same table is used twice, once to count subordinates (grouped by `managerId`) and once to look up manager names (matched by `id`).

### 32\. Sales Person — Easy

Problem| Solution  
---|---  
Find salespersons who have **no orders** linked to the company named `'RED'`. **DataFrames:** `sales_person(sales_id, name, ...)`, `company(com_id, name, city)`, `orders(order_id, date, com_id, sales_id, amount)`| Find sales_ids that sold to RED, then anti-join against all salespersons  
      
    
    import pandas as pd
    
    def sales_person(
        sales_person: pd.DataFrame,
        company: pd.DataFrame,
        orders: pd.DataFrame
    ) -> pd.DataFrame:
        # Step 1: Find company id for 'RED'
        red_id = company[company['name'] == 'RED']['com_id']
    
        # Step 2: Find sales_ids who sold to RED
        red_sales_ids = orders[orders['com_id'].isin(red_id)]['sales_id']
    
        # Step 3: Anti-join — exclude those sales_ids
        return sales_person[~sales_person['sales_id'].isin(red_sales_ids)][['name']]
    
    
    

**Key concept:** Multi-step filter: first identify the target company, then identify salespeople linked to it, then exclude them. This is a two-hop anti-join.

### 33\. Accepted Candidates From the Interviews — Medium

Problem| Solution  
---|---  
Find candidates who have `years_of_exp >= 2` AND `total_accepted_candidates >= 15`. **DataFrames:** `candidates(candidate_id, years_of_exp, interview_id)`, `rounds(interview_id, round_id, score)`| Filter candidates on experience, aggregate round scores, filter on total  
      
    
    import pandas as pd
    
    def accepted_candidates(candidates: pd.DataFrame, rounds: pd.DataFrame) -> pd.DataFrame:
        # Step 1: Filter candidates with >= 2 years experience
        eligible = candidates[candidates['years_of_exp'] >= 2]
    
        # Step 2: Sum scores per interview
        totals = rounds.groupby('interview_id', as_index=False)['score'].sum().rename(
            columns={'score': 'total_score'}
        )
    
        # Step 3: Join and filter on total score >= 15
        merged = eligible.merge(totals, on='interview_id')
        return merged[merged['total_score'] >= 15][['candidate_id']]
